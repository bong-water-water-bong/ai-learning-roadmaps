## What Makes a Great Contribution

We aim to maintain **signal over noise**. A great contribution is:

* **Relevant** to AI, ML, Deep Learning, or closely related fields
* **High quality** (clear, authoritative, and useful)
* **Up to date** (not deprecated or obsolete)
* **Well described** (concise explanation of what and why)
* **Non-promotional** (no personal ads, referral links, or spam)

---

## Ways to Contribute

>Please follow the existing structure when adding resources.

You can contribute to this project by:

**1. Adding New Resources:**

* Identify high-quality resources (articles, courses, books, videos, tools etc.) that align with the roadmap's structure and progression levels.
* Ensure the resources are credible and well-regarded in the AI/ML learning community. Prioritize free or affordable options whenever possible.
* Identify that your resources where comes fit.
* Improving existing entries (better descriptions, updated links)
* Follow the existing format in the README.md file for consistency. Include details like:
    * Resource title.
    * Type of resource (e.g., YouTube, Website, Blog, Course, etc.).
    * Link to the resource (URL).
    * Level (Beginner, Intermediate, Advanced)
* Organizing resources into better categories
* Adding learning roadmaps or study guides
* Proposing new sections via Issues

**2. Improving Existing Resources:**

* Fixing typos, broken links, or formatting issues
* Review the current resources and suggest corrections or updates if necessary.
* Provide feedback on the resource's effectiveness, clarity, or value for specific learning levels.
* Create a pull request (PR) or open an issue outlining your proposed changes or additions.


All contributions are reviewed by maintainers.

---

## Before Submitting a Pull Request

* **Thoroughly review** the README.md file to ensure your contribution aligns with the existing structure and content.
* **Search for existing issues** that might address your proposed addition or update.
* **Test any new links** you add to resources to verify they function correctly.
* **Write clear and concise commit messages** that describe your changes.

---


### Required Format


Maintain consistent formatting and indentation as used in the existing repository when adding a resource.

  **Example (Good)**

```md
- [Khan Academy – Probability](https://www.khanacademy.org/math/statistics-probability/probability-library)
```

>Example (Bad ❌)
```md
- Best AI Course Ever!!! – Learn AI fast and make money 💰
```

## Resource Quality Criteria
>If a resource fits multiple sections, choose the **most appropriate one**.

Please ensure the resource:

* Has **clear educational or practical value**
* Comes from a **credible author, institution, or organization**
* Is **accessible** (clearly mark if paid, free, or requires signup)
* Uses an **appropriate difficulty label** (Beginner / Intermediate / Advanced)
* Respects **copyright and licensing**

## Contribution Workflow

1. Fork this repository
2. Clone your fork

   ```bash
   git clone https://github.com/bishwaghimire/ai-ml-resources.git
   ```
3. Create a new branch

   ```bash
   git checkout -b add-resource-name
   ```
4. Make your changes
5. Commit with a clear, descriptive message

   ```bash
   git commit -m "Add transformer NLP course"
   ```
6. Push your branch

   ```bash
   git push origin add-resource-name
   ```
7. Open a [Pull Request](https://github.com/bishwaghimire/ai-learning-roadmaps/pulls)

---

## Pull Request Checklist

Before submitting your PR, confirm:

- [ ] Resource follows the required format
- [ ] Link works and is stable
- [ ] Description is neutral and concise
- [ ] No duplicate entries exist
- [ ] Section placement is correct

Incomplete or low-quality PRs may be closed without merging.
